根路径展示页

OutlookManager

用于集中管理 Outlook 邮箱账户、邮件公开分享、API 接入与日常运营排查。

主页作用 用来说明这个站点是做什么的，适合给访客、客户或内部同事先看一眼。
后台入口 管理员后台登录地址由部署者自定义，不会直接暴露在主页上。
公开分享 邮件公开分享可按需绑定独立域名，把公开页和后台分开。
当前主页说明
这里展示的是管理员在后台“设置”页填写的主页标题和主页介绍，用来告诉访问者这个网站的用途。
部署建议
如果启用了独立分享域名，公开邮件页会自动切换到指定域名，后台仍然保留在管理员入口地址。
集中管理 Outlook 账户
统一维护邮箱账户、分类、标签和健康状态，减少人工切换账号的成本。
公开分享指定邮箱页面
支持按邮箱生成公开访问页，可设置开关、有效期和访问密码。
提供 API 接口能力
可通过 API Key 接入外部流程，把邮箱管理、查询和标记接到自动化系统里。
公开主页
适合展示网站用途、服务范围和使用说明。修改内容请登录管理员后台后进入“设置”。
安全隔离
当前未启用独立分享域名，公开分享页默认使用当前站点域名。
管理员后台入口路径可在初始化时设置，也可在后台“设置”页面修改。根路径主页只负责介绍站点用途，不承担后台登录入口展示职责。
